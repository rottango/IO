GitHub to platforma do zarządzania projektem opartym na systemie kontroli wersji Git. Proces commitowania zmian polega na zapisywaniu wprowadzonych modyfikacji w kodzie źródłowym ,lub czymkolwiek innym jak plik, w formie commitów. Aby to zrobić, należy najpierw stageować zmienione pliki za pomocą polecenia git add, następnie utworzyć commit za pomocą git commit, a na koniec wysłać zmiany na serwer GitHub za pomocą git push. Każdy commit powinien być opatrzony opisem, który jasno i zwięźle opisuje wprowadzone zmiany. Można również dodawać pliki/kody źródłowe za pomocą interface'u graficznego na stronie GitHub. Szczególnie jest to proste, gdy nie działamy z kodem.
